Interpreter vs Compiled Languages
NLPL currently runs as an interpreted language on top of CPython. The ratios above reflect interpreter overhead versus ahead-of-time compiled code (C, Rust) and Python's highly optimized CPython interpreter.

Dispatch table speedup: A static class-level dispatch table + per-instance bound method cache replaced per-node regex processing, delivering 18.0x faster AST node dispatch. This is an interpreter-level optimization, not an algorithmic one.

Next steps to reduce NLPL/C ratio: LLVM native code generation backend (planned — Phase 3 compiler work) will eliminate interpreter overhead entirely, targeting <5x C performance for compute-intensive workloads.
