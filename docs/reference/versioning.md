# NLPL Versioning Policy

NLPL does not use version numbers during active pre-v1.0 development.

## Why no versioning yet?

The language is under heavy development. Features change, APIs evolve, and the compiler
pipeline is still being completed. Assigning version numbers to an incomplete project
creates false expectations and meaningless changelogs.

## When versioning will start

Versioning (semantic versioning, v0.1 through v1.0) will begin once the project reaches
a defined milestone. At that point a proper changelog will be written covering everything
from the beginning, organized as v0.1 through v1.0 to reflect the actual development arc.

## Current state

The project is pre-v1.0. Track progress via git commits and the README's status table.

## Future version plan (when the time comes)

Releases will follow semantic versioning starting at v0.1. The v0.x series will cover
the development period leading up to a complete, stable v1.0. Each version tag will have
a changelog written after the fact to accurately describe what was achieved.
